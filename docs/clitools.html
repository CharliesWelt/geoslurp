Command line tools¶

clitools/geoslurper.py¶

Program to download and manage Earth Science data

usage: clitools/geoslurper.py [-h] [-i] [-l] [--refresh] [--purge-cache [filter]] [--purge-data [filter]] [--purge-entry [filter]] [--pull [JSON]] [--register [JSON]]
                              [--local-settings LOCALSETTINGSFILE] [--config [JSON]] [--show-config] [--auth-config [JSON]] [--admin-config [JSON]] [--host hostname]
                              [--user user] [--add-user username] [--add-readonly-user username] [--password password] [--port port] [--usekeyring] [-v [VERBOSE]]
                              [--data-dir DIR] [--cache-dir DIR] [-d [PATTERN]] [--mirror [MIRRORALIAS]] [-f [PATTERN]]
-h, --help¶

Prints detailed help (may be used in combination with –dset for detailed JSON options)

-i, --info¶

Show information about selected datasets

-l, --list¶

List all datasets which are available to use

--refresh¶

Refresh the cache of the available datasets

--purge-cache <filter>¶

Purge the cache of the selected dataset. while optionally applying a filter for the files

--purge-data <filter>¶

Purge the data of the selected dataset. while optionally applying a filter for the files

--purge-entry <filter>¶

Purge the database entry of the selected dataset. Optionally applying a filter for the files

--pull¶

Pull data from online resource (possibly pass on options as a JSON dict

--register¶

Register data in the database (possibly pass on options as a JSON dict)

--local-settings <localsettingsfile>¶

Specify a different file to read the local settings from (default= ${HOME}/.geoslurp_lastused.yaml)

--config¶

Register user settings (pass as a JSON dict, e.g. {“DataDir”:”path/”})

--show-config¶

Show user configurations as stored in the database

--auth-config¶

Register (and encrypt on a user basis in the database) authentification services (pass as a JSON dict, e.g. {“servicename”:{“user”:..,”passw”:…}})

--admin-config¶

Register admin/default settings (pass as a JSON dict, e.g. {“DataDir”:”path/”})

--host <hostname>¶

Select host where the PostgreSQL/PostGIS server is running

--user <user>¶

Select postgresql user

--add-user <username>¶

Add a new postgresql user (you will be prompted for a password, or you can append the password after a colon e.g. pietje:secretpassword)

--add-readonly-user <username>¶

Add a new readonly postgresql user (you will be prompted for a password, or you can append the password after a colon e.g. pietje:secretpassword)

--password <password>¶

Select password for the postgresql user

--port <port>¶

Select the port where the database is served

--usekeyring¶

Set and get the system keyring to store the database password (alternatives are using –password or the environment variable GEOSLURP_PGPASS

-v, --verbose¶

Increase verbosity of the output one cvan use multiple v’s after another (e.g. -vv) to increase verbosity. The default prints errors only

--data-dir <dir>¶

Specify (and register) a dataset specific data directory DIR

--cache-dir <dir>¶

Specify (and register) a dataset specific cache directory DIR

-d <pattern>, --dset <pattern>¶

Select datasets or all datasets in a scheme (PATTERN is treated as a regular expression applied to the string SCHEME.DATASET)

--mirror <mirroralias>¶

Use a different mirror for prepending to relative filename uris, the default uses the mirror registered as ‘default’ in the database. A mirror can be registered in the database with –[admin-]config ‘{“DataMirrors”:{“MIRRORALIAS”:”MIRRORPATH”}}’.

-f <pattern>, --func <pattern>¶

Select geoslurp database functions or all functions in a scheme (PATTERN is treated as a regular expression applied to the string SCHEME.FUCNTION)
